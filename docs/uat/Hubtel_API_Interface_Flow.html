Unilove <> Hubtel API Interface Flow

Prepared: 2026-02-25 | Environment: https://unilove.iderwell.com

1. System Context

2. API Interface Matrix

Direction Purpose Endpoint Expected Result
Hubtel -> Unilove Programmable services interaction (USSD) POST /api/ussd/interaction Returns next USSD screen, release, or AddToCart response.
Hubtel -> Unilove Payment callback (Receive Money / Programmable Services) POST /api/payments/hubtel/callback Signature validated; payment persisted; order status updated.
Unilove -> Hubtel Initiate MoMo prompt POST /merchantaccount/merchants/{POS_SALES_ID}/receive/mobilemoney Pending/accepted response; final state via callback.
Unilove -> Hubtel Fallback status check GET /transactions/{POS_SALES_ID}/status?clientReference=... Final transaction status for reconciliation.

3. Callback Processing Controls

  • Header signature validated (`x-hubtel-signature` or fallback signature headers).
  • Payload normalized for multiple callback shapes.
  • Order resolved by client reference/session id.
  • Payment stored in `payments` table with raw callback payload.
  • Order state updated (`PAID` / `PAYMENT_FAILED`) and audited.

4. End-User Sequence (USSD)

  1. Customer dials shortcode and navigates menu.
  2. App returns `AddToCart` response to handover to Hubtel checkout.
  3. Hubtel prompts customer MoMo approval.
  4. Hubtel sends callback to Unilove callback endpoint.
  5. Order status updates in operations dashboard.
